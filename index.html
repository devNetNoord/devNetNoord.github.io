Serverless? Azure Functions? Hoe, Wat en vooral Waarom?

Jan de Vries / 16:00 - 17:00

De afgelopen jaren zijn er veel ontwikkelingen geweest in het maken van een schaalbare architectuur. Het is begonnen met het eenvoudig kunnen opschalen van virtuele machines in de cloud. Vlak daarna begon de PaaS revolutie en nu is het dan eindelijk tijd voor echte serverless oplossingen.

Bij serverless oplossingen hoef je je helemaal niet meer bezig te houden met het onderliggende server platform en kun jij je focussen op het leveren van waarde aan je klanten. Functions-as-a-Service (FaaS) is een belangrijk onderdeel in de serverless wereld. De Functions reageren vaak op triggers van buitenaf en handelen 1 specifieke actie af. Binnen Azure kunnen we hier onder andere Azure Functions voor gebruiken.

Tijdens deze sessie ga ik dieper induiken op wat Azure Functions nou eigenlijk zijn. Hoe je ze slim kunt inzetten binnen je eigen oplossingen en hoe de verschillende serverless oplossingen kunnen helpen bij het opzetten van een moderne architectuur.

Security in de pipeline. De shift naar Rugged DevOps.

René van Osnabrugge / 17:00 - 18:00

Ondanks wat vele bedrijven denken, is het niet zo dat je een DevOps team krijgt wanneer je developers en IT-Pro’s in een team stopt. Maar zelfs als ze wel goed met elkaar samen werken is er altijd nog een extra aspect waar rekening mee moet worden gehouden. Security. Security Officers die alle releases verfieren en goedkeuren en lange periodes met security tests behoren tot het verleden. Want als alles geautomatiseerd is en code en infrastructuur meerdere malen per dag worden uitgerold is het niet meer mogelijk om deze tests handmatig uit te voeren.

Maar hoe kan je er toch voor zorgen dat je applicatie, infrastructuur en ook je pipeline secure is? Rugged DevOps gaat over het secure maken en houden van je gehele proces en applicatie en zorgt ervoor dat security een geintegreerd onderdeel is van het geheel.

In deze sessie zal ik je door de concepten van Rugged DevOps leiden, de risico’s die bedrijven vandaag de dag lopen en zal ik praten over een aantal strategieen en tools die je kunnen helpen om security in je pipelines te integreren.

ARM Templates; Infra-as-Code en VSTS vanuit een developer perspectief

Peter Rombouts / 19:00 - 20:00

In deze sessie wordt uiteengezet hoe om te gaan met Infra-as-Code vanuit een ontwikkelaars perspectief. Vanuit een real-life scenario worden lessons learned getoond, en hoe deze toegepast zijn in Azure. Tevens komen enkele zelf of mede ontwikkelde VSTS extensions voorbij, welke het CICD proces vergemakkelijken. Ook ga ik in op hoe deze extensies relatief snel kunnen worden ontwikkeld en uitgerold.

Doelgroep: IT Pro en Developers
Takeaway: Inzicht in hoe CICD, Infra-as-Code, en Development samen kunnen werken.

Blazor, cutting edge or an accident waiting to happen?

Roelant Dieben / 20:00 - 21:00

Blazor belooft ons full stack web development in de browser door gebruik te maken van Webassembly, maar hoe serieus is deze belofte? Tijdens dit uur bekijken we de (on)mogelijkheden van Blazor, duiken we dieper in de werking van Blazor en Webassembly en bekijken we hoe ze dit voor elkaar hebben gekregen en hoe wij deze techniek kunnen gebruiken om geweldige applicaties te maken.
