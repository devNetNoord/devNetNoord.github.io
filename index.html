Angular front-end zonder back-end (serverless computing met Azure Functions)

door Richard de Zwart en Pieter Leijenaar van Get There

De gedachte achter “serverless” is meer “worry about the server less”. Alle grote cloud providers bieden varianten op dit thema: Azure Functions, Amazon Lambdas, Google Cloud Functions. Een klein stukje code met slechts één duidelijke taak (nano-service?) dat geheel beheerd wordt door de cloud provider. Je hebt geen VM, je hebt geen AppService, je hebt geen idee waar het draait. Je weet alleen maar dat, zodra het nodig is, jouw functie onbeperkt geschaald wordt. En dat je eenvoudig toegang hebt tot Message Queues, HttpRequests, Databases, WorkFlows, etc.
We laten zien hoe je een SPA bouwt in Angular 2 en die koppelt aan een API die geheel met Azure Functions is opgebouwd. Enige affiniteit met Star Wars is gewenst.

Product development using state-of-the-art techniques and technologies

door Hans Gringhuis van Magister

Magister is marktleider in educatieve en gebruiksvriendelijke software voor het voortgezet onderwijs. In 15 jaar is Magister uitgegroeid tot een product waar meer dan 500 scholen, met hun docenten, leerlingen en ouders, dagelijks gebruik van maken. Achter deze marketing zinnen schuilt een hele “software wereld”. Hoe ga je om vanuit software evolutie en technologie perspectief om met een code base van 15 jaar waarbij je er altijd voor moet zorgen dat tijdens de verbouwing de winkel openblijft? Het dagelijks gebruik van het platform door vijfhonderd scholen met hun docenten, leerlingen en ouders -in totaal bijna 3 miljoen gebruikers- zorgt voor interessante uitdagingen op het gebied van security, performance en schaalbaarheid. Om je een indruk te geven, het Magister platform verwerkt enkele duizenden API calls per seconde. Naast een grote hoeveel calls, zijn de gebruikers ook in staat om een indrukwekkende hoeveelheid data (lees een paar honderd TB) te genereren.
In de presentatie laten we zien hoe we vanuit een software engineering perspectief invulling geven aan de ontwikkeling van het Magister platform. Onderwerpen die aan bod komen zijn: het gebruik van microservices, de toepassing van DDD, het gebruik van het Microsoft platform met de eerste stappen naar de (hybride)cloud en de gebruikte (gedistribueerde) software architecturen.

Infuse some AI - Making apps a bit smarter

door Hans Boelman van Ordina

Wouldn't it be great if an application could tell you what it recognizes on the things you upload, who it identifies on the pictures, tells what it sees, make smart crops, filter out negative comments and recognizes gestures? In this talk I will introduce you to the world of the Microsoft Cognitive Services and show you how to use them to extend applications. We dive into some use-case scenarios and I will show how easy it is to integrate them with other Azure services like Functions, Cosmos DB and Azure Search. You will see demos and walk away with ideas on how to use this in your own applications.

The Lazy Dev’s guide to doing Identity the cloud native way

door Annejan Barelds van Sogeti

"Real dev's don't like security tokens." De meeste developers zijn geen hardcore identity & access-enthousiastelingen zijn die de hele week aan gebruikersauthenticatie willen werken. En daarom laat Annejan je in deze sessie zien hoe je Microsoft Azure het werk kunt laten doen met Azure Active Directory en Azure AD B2C! De invalshoek van deze sessie is om aan de hand van specifieke scenario's te bespreken hoe Azure AD het identity-probleem voor jou oplost, zodat jij je tijd kunt besteden aan je echte werk: je klanten helpen met innovatieve oplossingen. Geen droge securitysessie dus, maar tools voor in de cloud development toolbox:
• Ben je al aan het experimenteren met een microservices-architectuur? En heb je al een oplossing voor identity & access in een services-omgeving?
• Werk je aan applicaties die door consumenten gebruikt moeten kunnen worden? Worstel je ook met de eigenaardigheden van consumer identities?
• Heb je te maken met zowel traditionele webapps, SPA's, native apps, en machine-2-machine communicatie? En weet jij voor elk scenario het juiste authenticatiemechanisme te kiezen?
Deze en andere onderwerpen zullen de revue passeren, zodat jij in een uur klaar bent voor Identity en Access in de cloud.

IoT, Machine Learning en Industry 4.0 in de praktijk

door Rik Opdam van Ilionx

Kunstmatige intelligentie zal fundamentele veranderingen brengen in de manier waarop wij leven, werken en met elkaar omgaan. Maar ook hoe wij produceren. Momenteel staan wij aan het begin van de 4e technologische revolutie. Kunstmatige intelligentie en cloud technologie spelen een belangrijke rol in deze verandering, maar hoe werkt dat nu in de praktijk?
De productie van huishoudelijke apparaten is een waar ambacht. De afmeting en samenstelling van onderdelen is zo'n precisiewerk, dat afwijkingen amper of zelfs niet waarneembaar zijn met het met het blote oog. Of een onderdeel aan het eind van de productielijn wordt goedgekeurd of afgekeurd, dat is voor een belangrijk deel afhankelijk van de manier waarop de machines zijn ingesteld.
Dit is een samenspel van mens en machine; operators zorgen ervoor dat de machines de juiste instellingen hebben om ervoor te zorgen dat de uitval zo laag mogelijk is. Ook hierbij wordt de mens steeds meer bijgestaan door machines. Dankzij machine learning is het namelijk mogelijk om steeds nauwkeuriger te voorspellen welke onderdelen van het productieproces mogelijk zorgen voor een afwijkingen.
Voor een internationale elektronicafabrikant richtte ilionx onlangs zo'n machine learning-programma in. Het doel: bepalen welke parameters in het productieproces moeten worden aangepast om de uitval zo klein mogelijk te maken.

Shifting gears for .NET: Moving to containers for .NET developers

door Alex Thissen van Xpirit

The landscape of distributed applications is rapidly changing from server-based to using clusters of containers and server-less computing. .NET is capable of creating such cross-platform applications, being run in Docker containers, but it is not trivial to get started with this. This session will focus on providing an overview of the changing world for .NET developers, with a new workflow that has Docker and containers at its core. You will learn how Visual Studio 2017, .NET Core and Docker play a strategic role in building, packaging, deploying and running modern .NET applications, but also what it takes to develop these apps on your laptop and in container clusters.

Hands-on Azure Functions

door Richard de Zwart

In de demo heb je gezien hoe je een stukje C#-script in het Azure Portal in een Function kunt hangen. Snel voor elkaar, maar dat is natuurlijk niet hoe je het in de praktijk wilt doen. In deze hands-on ga je code schrijven in C# in Visual Studio en die automatisch deployen naar Azure. Je krijgt daarmee een idee van het gemak -en de ongemakken- van het maken en onderhouden van Azure Functions. Een prima basis om voor jezelf te bepalen of dit in de dagelijkse praktijk bruikbare technologie is of niet.

Benodigd: laptop met een up-to-date VS2017, minimaal .NET framework 4.6.2, Azure Functions Tools extension voor VS2017.
